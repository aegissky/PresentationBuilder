# LLM 프롬프트 표준 (PROMPT-STANDARDS)

> PresentationBuilder가 LLM(Claude / ChatGPT / Cursor / 기타)을 호출할 때의 표준 프롬프트 형식.
> 본 문서는 `INHERITANCE.md::PB-PROMPT-STANDARD` 규칙의 SSOT.

---

## §1. 적용 범위

| 시나리오 | 프롬프트 표준 적용 |
|---------|----------------|
| 새 PPT 부트스트랩 후 첫 트리거 | 필수 |
| 발표 생성 도중 페이지 추가 | 필수 |
| 스크립트(MD) 톤 조정 | 권장 |
| 단순 텍스트 편집 | 면제 |

---

## §2. 진입 프롬프트 (7-항목 인터뷰)

LLM이 사용자에게 다음 형식으로 **한 블록으로** 질의해야 한다 (한 항목씩 묻지 말 것):

```
📋 발표 자료 생성 준비 — 7가지를 알려주세요.
키워드만 입력해도 됩니다.

① 발표 유형
   착수보고 / 제안서 / 사업계획서 / 중간보고 / 완료보고 / 교육자료 / IR피치덱 / 기타

② 조직명 · 프로젝트명
   예) 웹케시 — 통합자금관리시스템 고도화

③ 청중
   임원(C-level) / 실무책임자 / 현업담당자 / 고객사 / 투자자 / 일반

④ 문체·분위기
   공식·신뢰감 / 전문·간결 / 친근·설득 / 역동·자신감 / 학술·분석

⑤ 전문 용어 수준
   하(누구나 이해) / 중(업계 용어 허용) / 상(기술 전문 용어 자유)

⑥ 총 발표 시간 (분)
   10 / 20 / 30 / 45 / 60 / 90

⑦ 핵심 메시지 3개
   예) "안정적 이전", "업무 자동화", "비용 절감 30%"

──────────────────────────────────────
선택 옵션:
  목차 구성이 있으면 붙여 넣어주세요 (없으면 자동 제안)
  테마: navy(기본) / forest(녹색) / charcoal(중후)
```

미입력 시 default:

| 항목 | default |
|------|---------|
| 발표 유형 | "보고서" |
| 청중 | "임원" |
| 분위기 | "공식·신뢰감" |
| 전문 용어 | "중" |
| 발표 시간 | 30분 |
| 테마 | "navy" |

## §3. TOC 확정 체크포인트 (필수 게이트)

Phase 3 종료 시 LLM은 다음 형식으로 사용자 확인을 받아야 한다. **확인 없이 Phase 4 진행 금지** (PB-PAGE-SCRIPT-SYNC 위반 방지):

```
[📋 TOC 확인]
아래 목차로 생성합니다. 수정하시면 알려주세요.

총 {N}페이지 · 예상 {M}분 · 테마: {theme}
청중: {audience} · 분위기: {tone} · 전문도: {difficulty}

── Ⅰ. {섹션 제목} ({n}페이지, {m}분) ──
  {pageId}  {페이지 제목}  [{레이아웃 ID}]  {예상 초}초
  ...

── Ⅱ. {섹션 제목} ({n}페이지, {m}분) ──
  ...

→ "확인합니다" 또는 수정 내용을 말씀해주세요.
```

## §4. 페이지 생성 표준 (PB-PAGE-SCRIPT-SYNC)

각 페이지 생성 시 LLM은 4종을 **동시에** 작성해야 한다. 한 종이라도 누락되면 헌법 위반:

| 산출물 | 경로 | 의무 |
|--------|------|------|
| HTML 슬라이드 | `<pageId>.html` | 필수 |
| 발표 스크립트 | `script-text/<pageId>.md` | 필수 (⏱·🎯·💡 3섹션 포함) |
| viewer 갱신 | `script-text/viewer.html` `SCRIPTS`·`MD[]` | 필수 |
| README 누적 | `script-text/README.md` 표 행 | 필수 |

상세 형식: [`_GUIDE/SKILL.md`](_GUIDE/SKILL.md) §4-1 ~ §4-6

## §5. 완료 보고 형식 (Phase 7)

```
✅ {프로젝트명} 발표 자료 생성 완료

📁 {output.path}
📊 {N}개 페이지 · 예상 {M}분 · {theme} 테마

생성 파일:
  HTML 슬라이드:    {N}개
  발표 스크립트 MD:  {N}개
  공통 모듈:        복사 완료
  스크립트 뷰어:    script-text/viewer.html
  목차:             index.html
  발표 진입점:      index01.html

품질 체크: {통과한 항목 수}/{전체 항목 수}

🎯 권장 다음 단계:
  1. index.html → 전체 흐름 확인
  2. 수정 필요 페이지 → 편집 모드(Esc)로 인라인 수정
  3. script-text/viewer.html → TTS 시청 + 스크립트 다듬기
  4. TTS/audio/ → 직접 녹음 파일 추가 (선택)
  5. index01.html → 최종 발표 리허설
```

## §6. 자주 쓰는 사용자 트리거 예시

### A. 풀 자동 (기본값 사용)
```
발표 만들어줘
```

### B. 메타정보 함께
```
발표 만들어줘.
조직: ACME / 유형: IR피치덱 / 청중: 투자자 / 시간: 15분
핵심: "시장 폭발", "검증된 솔루션", "8x 수익 성장"
```

### C. 자료 위치 명시
```
_input/01_documents 에 RFP 응답 초안이 있어. 이걸 기반으로 제안서 만들어줘.
청중은 고객사, 톤 공식·신뢰감, 25분.
```

### D. 단일 페이지 재생성
```
02-3 페이지를 L11(아키텍처)로 다시 만들어줘.
이번엔 인프라 레이어를 4계층으로 분해해서 보여주는 다이어그램 중심으로.
```

### E. 톤 일괄 조정
```
script-text/*.md 모든 스크립트의 톤을 "친근·설득"으로 다시 써줘.
슬라이드 본문은 건드리지 마.
```

## §7. LLM 도구별 진입 보조

### Claude Code (CLI)
- 작업 디렉터리에서 `claude` 실행 → 본 폴더의 `CLAUDE.md` + `_GUIDE/SKILL.md` 자동 인식
- 트리거: 자연어 그대로

### ChatGPT / Claude.ai 웹
1. 첫 메시지에 `_GUIDE/SKILL.md` 전체 첨부
2. `PROMPT-STANDARDS.md` 함께 첨부
3. `_input/` 자료 드래그·드롭

### Cursor / Continue / Windsurf
1. 본 폴더 열기
2. `@file _GUIDE/SKILL.md` + `@file PROMPT-STANDARDS.md` 컨텍스트 핀
3. Agent 모드 활성화 후 트리거

### API 직접 호출
- System Prompt: `_GUIDE/SKILL.md` 전체 + 본 문서
- User: §6 예시 형식

## §8. 금지 사항

| 금지 | 사유 |
|------|------|
| 7-항목 인터뷰를 1항목씩 묻기 | 사용자 피로도 폭증, 시간 낭비 |
| TOC 확인 없이 Phase 4 직행 | 재작업 비용 큼 |
| HTML만 생성하고 script-text 누락 | PB-PAGE-SCRIPT-SYNC 위반 |
| 자료 분석 없이 콘텐츠 임의 생성 | 사실 왜곡 위험 |
| 5개 이상 옵션 질문 | 사용자 결정 부담 |
